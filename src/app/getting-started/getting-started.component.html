Getting started

Welcome to Micro! We want to help you 10x your development experience by providing one account to access a catalog of public APIs for everyday use. We hope you'll never have to signup to another third party API provider again. Follow this getting started guide to get familiar with the platform.

Sign up

Sign up or log in to Micro so you can get a token. You can use password based log in or oauth. However, after logging in with oauth, your password is no longer valid unless you reset it (for security reasons).

Discover APIs

The explore page (https://m3o.com/explore) contains all APIs present on Micro. The full text search includes service name, endpoint names and the readme.

Getting an API token

To call Micro APIs, you need an API token. A default token is autogenerated after you sign in and stored as a cookie, so you can use it to access APIs via Query pages in the UI (example: https://m3o.com/helloworld/query). Otherwise you'll need to generate API tokens for external use.

To create a new API token, visit https://m3o.com/settings/keys. Specify which APIs the given token can call or leave scopes blank to create a token that can call any APIs.

How to use a Micro API

Micro APIs can be accessed via the url `https://api.m3o.com/v1/`. The specific api and endpoint name are appended to the url e.g https://api.m3o.com/v1/helloworld/call. All API calls require an API token provided via the `Authorization: Bearer` header. Our APIs provide a JSON based request/response format so also require a `Content-Type: application/json` header.

After familiarising yourself with an API by reading the API reference (for example: https://m3o.com/db#Read) you probably want to start calling the API. Micro APIs can be called via HTTP in any language or using our node.js and Golang specific clients.

Examples can be found under the usage tab: https://m3o.com/db#Read-usage.

Pricing

The majority of Micro APIs are free or very low cost. Third party APIs or calls that do specific things like SMS tend to cost more.

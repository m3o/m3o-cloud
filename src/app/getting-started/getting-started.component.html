Getting Started

Welcome to Micro! Follow this getting started guide to get familiar with the platform.

Sign up

Sign up or log in to Micro so you can get a token. You can use password based log in or oauth.

Note: For security purposes, if you log in with oauth, the password is no longer valid unless you reset it.

Discover APIs

Browse all our public APIs on the explore page (https://m3o.com/explore).

Generate an API token

To call Micro APIs, you'll need an API token. To create a new API token, visit https://m3o.com/settings/keys. Specify which APIs the given token can call or leave scopes blank to create a token that can call any APIs.

Using an API

Micro APIs can be accessed via the url `https://api.m3o.com/v1/`. The specific api and endpoint name are appended to the url e.g https://api.m3o.com/v1/helloworld/call. All API calls require an API token provided via the `Authorization: Bearer` header. Our APIs provide a JSON based request/response format so also require a `Content-Type: application/json` header.

After familiarising yourself with an API by reading the API reference (for example: https://m3o.com/db#Read) you probably want to start calling the API. Micro APIs can be called via HTTP in any language or using our node.js and Golang specific clients.

Examples can be found under the usage tab: https://m3o.com/db#Read-usage.

Billing

The majority of Micro APIs are free during the beta period. Third party APIs or calls that perform specific functions such as SMS or email are charged for and billed per request. Each account has a credit based balance. Any request made to a paid API will be debited from the balance in real time. When your balance is zero requests to paid APIs will return a blocked status. To check your balance and top-up your account head to the billing page at https://m3o.com/overview/billing

Getting Started

Welcome to Micro! Follow this getting started guide to get familiar with the platform.

Sign up

Sign up or log in to Micro so you can get a token. You can use password based log in or oauth.

Note: For security purposes, if you log in with oauth, the password is no longer valid unless you reset it.

Discover APIs

Browse all of our public APIs on the Explore page.

Generate an API token

To create a new access token, visit API settings page. You can specify which APIs the token can call or leave scopes blank to create a token that has access to all the APIs.

Using an API

Micro APIs can be accessed via the url "https://api.m3o.com/v1/[service]/[endpoint]". All API calls require an API token provided via the "Authorization: Bearer" header. Our APIs provide a JSON based request/response format and also require a `Content-Type: application/json` header for each request.

After familiarising yourself with an API by reading the API reference (for example: https://m3o.com/db/api) you probably want to start calling the API. Micro APIs can be called via HTTP in any language or using our node.js and Golang specific clients.

Billing

The majority of Micro APIs are free during the beta period. Third party APIs or calls that perform specific functions such as SMS or email are charged for and billed per request. Each account has a credit based balance. Any request made to a paid API will be debited from the balance in real time. When your balance is zero requests to paid APIs will return a blocked status.

To check your balance and top-up your account head to the billing page at https://m3o.com/overview/billing

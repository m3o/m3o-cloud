Getting started

Welcome to Micro! We want to help you 10x your development experience by providing one account to access a catalog of public APIs for everyday use. We hope you'll never have to signup to another third party API provider again. Follow this getting started guide to get familiar with the platform.

Sign up

Sign up or log in to Micro so you can get a token. You can use password based log in or oauth. However, after logging in with oauth, your password is no longer valid unless you reset it (for security reasons).

Discover APIs

The explore page (https://m3o.com/explore) contains all APIs present on Micro. The full text search includes service name, endpoint names and the readme.

Getting a token

To call Micro APIs, you need tokens. We autogenerate a token after you sign in that is saved as a cookie, so you can go to a Query page (example: https://m3o.com/helloworld/query) and make calls straight away.

If you want to create new tokens, visit https://m3o.com/settings/keys. Specify which APIs the given token can call or leave scopes blank to create a token that can call any APIs.

How to use a Micro API

Micro currently only provides secret API tokens. That means you have to use those tokens from a backend. There is currently no way to use it in a Firebase-like fashion, where access control rules govern API calls.

After you familiarised yourself with an API by reading the API reference (for example: https://m3o.com/db#Read) you probably want to start calling the API from your backend. Currently there are a few generic Micro clients in different languages. By generic, we mean a simple client that can call any service on m3o.com. Type safe/annotated for specific APIs are not available yet but they are under development.

Examples for clients in different languages can be seen under the usage tab: https://m3o.com/db#Read-usage.

Pricing

Most of the Micro APIs are free or very low cost. Third part APIs or calls that do specific things like SMS tend to cost more.
